# 3.2–3.4 Нефункциональные требования

## Удобство использования
- простой UI (SwiftUI)
- минимальное количество действий
- адаптивность iPhone/iPad
- обработка ошибок ввода

## Надежность
- хранение данных в SQLite
- сессия в UserDefaults
- работа без интернета (кроме курса валют)
- обработка ошибок без краша
- восстановление состояния после перезапуска

## Производительность
- запуск ≤ 3 сек
- загрузка счетов ≤ 1 сек
- карта ≤ 2 сек
- поиск отделения ≤ 1 сек
- плавный UI 60 FPS
